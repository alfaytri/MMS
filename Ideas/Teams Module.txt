Teams Module — Complete Technical Breakdown
1. PAGES
1A. Team & Employee Management (/teams → TeamsEmployees.tsx)
Header Bar:

Title: "Team & Employee" with 3 summary badges: {n} teams, {n} employees, {n} vehicles

Compact view: vertical list of single-row entries

Right Panel — Employee Pool Sidebar (EmployeePoolSidebar, 264px):

Vehicle Pool (VehiclePool): shows unassigned vehicles; drag target to return vehicles to pool

Data Hooks:

useTeams() — fetches teams table (soft-delete filtered)
useEmployees() — fetches employees + employee_services junction for skills
useVehicles() — fetches vehicles table
useSchedules() — fetches schedules table
useScheduleAssignments() — fetches team_schedule_assignments
useActivityLog() — last 200 entries from activity_log
useTraccarDevices() — GPS device records
useToolAssetCounts() — tool/asset inventory counts per team and employee
Schedule Resolution Logic (in useEffect):

On load, for each team, finds the active team_schedule_assignment where startDate ≤ today ≤ endDate
If the resolved schedule_id differs from the team's stored schedule_id, auto-updates the DB
Uses a resolvedTeamIds ref Set to prevent re-processing
Handlers (from useTeamHandlers hook):

handleRemoveEmployee(empId, teamId) — unassigns employee from team
handleDropEmployee(empId, teamId) — assigns employee to team via drag
handleDropLeader(empId, teamId) — sets employee as team leader
handleRemoveLeader(teamId) — removes leader assignment
handleDropVehicleToTeam(vehId, teamId) — assigns vehicle to team
handleRemoveVehicle(teamId) — unassigns vehicle
handleDropVehicleToPool(vehId) — returns vehicle to pool
handleDropToStatus(empId, status) — changes employee status
handleCreateSchedule / handleEditSchedule / handleDeleteSchedule
handleAttachSchedule / handleDetachSchedule
handleArchiveEmployee / handleArchiveTeam / handleArchiveVehicle — soft-delete

5. KEY PATTERNS
Drag & Drop: HTML5 native drag via dataTransfer.setData("text/employee-id", id). Separate data keys for employees vs vehicles. Drop zones on: team cards, leader slots, vehicle slots, status tabs, vehicle pool.
Soft Deletes: All entities use deleted_at timestamp. Queries filter .is("deleted_at", null).
Audit Trail: Every create/edit/archive/assign/remove action calls audit() with entity info + old/new data diff.
Skill Resolution: Employee skills stored in employee_services junction table, loaded in useEmployees() with a second query + Map-based merge. Team skillset is aggregated from all members at display time.
Schedule Resolution: Active schedule determined by date-range overlap on team_schedule_assignments, auto-synced to teams.schedule_id
