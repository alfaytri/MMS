Contact Center Module
Overview
The Contact Center is a persistent sidebar (w-72) that lives across all pages. It's the unified hub for live phone calls (3CX), WhatsApp messaging (dual-provider: Whapi + WATI), customer CRM, address management, agent resources, and task queues.

1. Layout & Navigation
File: ContactCenterSidebar.tsx

The sidebar has 3 states:

State	Width	Content
Collapsed	w-10	Mini icon bar (Phone, User, MapPin, Package, History, Chat icons). Click ChevronRight to expand.
Chat List	w-72	Dialer + conversation list + tabs. Default view.
Customer Detail	w-72	Dialer + CRM header + scrollable CRM/Address/Products + Thread + Orders + Chat Input (pinned bottom).
State is managed by view: "chat-list" | "customer-detail" and collapsed: boolean.

13. State Management Architecture
File: useContactCenterState.ts (Orchestrator, 332 lines)

Composes 8 sub-hooks:

Sub-Hook	File	Responsibility
useDialerState	useDialerState.ts	Call state machine (idle→calling→connected→idle), duration timer, WA mini-chat
useCustomerData	useCustomerData.ts	Customer lookup (phone→customer), CRM editing, unknown caller flow, pending payments
useChatMessages	useChatMessages.ts	Chat input, attachments, source selection, reaction sending, file upload to chat-media bucket
useAddressState	useAddressState.ts	Blue Plate / Google Coords CRUD, coordinate auto-fetching
useBlockAutoMessages	useBlockAutoMessages.ts	Block/unblock customer, auto-message stop toggles
useWhatsAppWindow	useWhatsAppWindow.ts	24h window status calculation from last inbound message time
useLiveConversations	useLiveConversations.ts	Real-time conversation list from chat_conversations with Supabase Realtime subscription
useLiveThread	useLiveThread.ts	Real-time message thread for selected conversation + active_agent_calls tracking
Extracted Action Modules:

Module	File	Actions
ccCallActions	ccCallActions.ts	handleCallAction (dial via 3CX API, create active_agent_calls row, insert local call message), handleHangupAction, handleAnswerAction, handleRejectAction
ccMessageActions	ccMessageActions.ts	handleSendChatMessageAction (dual-provider send: Whapi for whatsapp, WATI for whatsapp_api, with attachment upload, reply quoting, delivery status tracking)
14. 3CX Integration
Files: use3cx.ts, use3cxCallPoller.ts, use3cxWebSocket.ts, useCurrentUserExtension.ts, usePhoneLines.ts

Hook	Purpose
use3cx	Wraps api-3cx edge function calls: makeCall(dn, dest), callAction(dn, pid, action), getParticipants(dn)
use3cxCallPoller	Polls participant status every 2s during calls. Detects phase transitions: ringing → connected → disconnected
useCurrentUserExtension	Resolves current user's 3CX extension from app_settings (key: 3cx_extensions, maps profile_id → extension)
usePhoneLines	Fetches phone_lines table rows ordered by sort_order
Background Polling: Every 3 seconds when idle, checks for incoming calls via getParticipants. If a ringing participant is found → triggers incoming call UI.

Call Lifecycle:

Agent clicks Call → cx_makeCall(extension, destination) → state = calling
Poller detects connected → state = connected, start duration timer
Creates active_agent_calls row (agent_id, customer_id, direction, started_at)
Inserts local chat_messages row with external_id: '3cx_local_{timestamp}'
Hangup → cx_callAction(dn, pid, 'drop') → updates active_agent_calls.ended_at
cx_call_journal trigger fires → enriches/replaces local message with real duration, recording info
15. WhatsApp Dual-Provider Architecture
Message Sending:

Source	Provider	Edge Function	Reply Support	Reactions
whatsapp	Whapi	api-whapi	✅ (quoted message ID)	✅
whatsapp_api	WATI	api-wati	❌	❌
External ID Prefixes: whapi_{id}, wati_{id}, 3cx_local_{ts}, 3cx_journal_{uuid}, agent_{ts}_{random}

24h Window: Only enforced for WATI (whatsapp_api). Calculated from last inbound message timestamp in useWhatsAppWindow. When closed, free-text input is disabled and only template buttons are shown.

16. Contact Center Context
File: ContactCenterContext.tsx

A React Context that publishes:

selectedCustomer (id, name, phone) — consumed by other modules (e.g., Create Visit pre-fills customer)
selectedAddresses — consumed for address auto-fill
phoneLookupHandler — allows external modules to trigger customer lookup by phone
customerLookupHandler — allows external modules to open customer detail by ID
17. Database Tables
Table	Purpose
chat_conversations	Conversation metadata: customer_id, channel, last_message, unread_count, team_id
chat_messages	Individual messages: text, from_type, source, attachments (JSONB), call_metadata (JSONB), external_id, delivery_status, reactions
active_agent_calls	Live call tracking: agent_id, customer_id, direction, started_at, ended_at
cx_call_journal	3CX CDR records: external_number, agent_name, duration, timestamps. Trigger process_cx_call_journal auto-creates/enriches chat messages
contact_center_tasks	Agent task queue: visit_id, customer_id, task_type, status, notes
phone_lines	Division phone lines: label, number, cx_dn, is_emergency, sort_order
agent_resources	Knowledge base files: title, type, file_url, category
agent_qa	Q&A entries: question_en/ar, answer_en/ar, category
customer_addresses	Address storage: label, line, type, coords, local_address (JSONB)
customer_phones	Multi-phone: phone, is_primary, customer_id
18. Key Features Summary
Feature	Implementation
Live 3CX Dialer	Full softphone: dial, answer, reject, hangup via API. Background polling for incoming calls
Dual WhatsApp	Whapi (personal numbers, reactions, replies) + WATI (business API, templates, 24h window)
Unified Thread	Calls + WhatsApp + SMS merged chronologically per customer
Unknown Caller	3-step flow: Prompt → Attach to existing OR Create new
Customer CRM	Inline editing of name, type, multi-phone, block/unblock with evidence images
Address Management	Blue Plate (Qatar format) + Google Coords with auto-geocoding, draggable to other modules
Agent Resources	Searchable Q&A + file library with insert-to-chat and send-to-customer
Task Queue	Real-time task cards (cancel/reschedule/follow-up/unavailable) with resolve/dismiss
Products & Warranty	Live warranty countdown with color-coded indicators
Order History	Last 20 orders with status badges + quick backwork/new order actions
Delivery Status	Per-message tick marks (✓ sent, ✓✓ delivered/read, ✗ failed)
Reactions	Emoji reactions on Whapi messages with aggregated display pills
Reply Quoting	Reply-to preview bar with quoted message context
Auto-Message Control	Per-customer toggle to stop all reminders or disable individual ones
Permission Gating	Emergency-only agents get read-only chat (cannot send messages)
