3.1 Map initialization (one-time effect, deps [])

const map = L.map(mapContainerRef.current, { zoomControl: false })
  .setView([25.2854, 51.5310], 12);   // Centered on Doha (~West Bay), zoom 12
L.tileLayer("https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png", {...}).addTo(map);

Two L.layerGroup() are created and stored in refs:

5. PERFORMANCE NOTES
Icon caching prevents L.divIcon allocation per render — only 8 icon objects total exist regardless of fleet size.
useMemo on filtered list — avoids re-filtering on every map update.
Layer-group clearLayers() + add is faster than individually removing markers.
One map instance — guarded by if (mapRef.current) return in init effect.
Tile attribution still rendered (OSM legal requirement).
6. EDGE CASES & GOTCHAS
Leaflet default icon paths are patched at module load to load from cdnjs.cloudflare.com (workaround for Vite asset resolution of Leaflet's bundled marker images). Without this, default markers would 404.
Map height = parent height — relies on the wrapper having h-full. If you embed it elsewhere without giving it height, the map will be 0px tall.
zoomControl: false means there's no UI to zoom — users can only pinch/scroll-wheel zoom or trigger zoom via "select team" (forces zoom 15).
Sidebar search ≠ map filtering — hiding a team from the sidebar list does not remove its marker.
showOrders doesn't filter by status — it's a single on/off for the entire order layer.
No realtime subscription in this file — refresh is manual (refetch button) or whatever interval useTeamLocations uses internally.
Stopped status uses raw Tailwind bg-orange-500 instead of a semantic token — minor design-system violation worth fixing.
Marker popup HTML uses inline-styled <p> tags with hex colors (#64748b, #0ea5e9) — these will not respect dark-mode theming.
