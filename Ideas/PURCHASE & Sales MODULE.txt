Reason text, restock warehouse selector, notes
Submit: creates unified return record
Shared Sales Dialogs (SoDialogs.tsx):
SoPaymentDialog:

Amount, Method (8 payment methods), Reference, Notes
Validates amount > 0
Creates payments record with source_type: sale_order
Audit log entry
SoDeliveryDialog:

Warehouse selector
Per-item qty inputs (max = ordered - already delivered)
Submit: calls deduct-sale-stock edge function → atomic FIFO deduction → COGS entry → stock movement → stock level update
Auto-updates SO status: partially_delivered or delivered
Generates delivery number sequence
Edge Functions (Backend):
deduct-sale-stock — Atomic FIFO deduction on delivery

Checks available stock per warehouse
Calls deduct_fifo_layers RPC (row-level locking)
Creates stock movement, COGS entry
Updates stock_level + releases reserved_qty
Recalculates average_cost
process-receival — Creates FIFO layers on purchase receival

Creates cost layers per brand variant
Records stock movements
Updates stock_level
Recalculates average_cost
reserve-stock — Reserves stock on SO confirmation

Increments reserved_qty on brand variants
Returns warnings for insufficient stock
