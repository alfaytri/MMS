Memoized derived value

const activeDivisions = useMemo(() => {
  if (activeDivisionIds !== null) return activeDivisionIds;
  if (idEntries.length > 0) return new Set(idEntries.map(d => d.id));
  return new Set<string>();
}, [activeDivisionIds, idEntries]);
Lazy-init pattern: until the user touches a division chip, it's "all divisions" computed from idEntries. Avoids a flash of empty calendar while divisions load.

What TeamCalendar renders (high-level)
A grid: rows = teams, columns = hours (e.g. 7am–10pm or Ramadan extended hours via useCalendarSchedule).
Each scheduled visit becomes a colored block placed at its time slot, span = duration.
Block color = visit type color; left-border or chip = division color.
Block content: time, customer name (clickable), service summary, action icons (✎ edit, ⇄ swap) gated by permissions.
Empty rows = teams with no work that day.
"Fit to View" mode collapses the whole day's hour range to fit within container width.
7. SWAP TEAM DIALOG — <SwapTeamDialog>
<SwapTeamDialog order={swapOrder} onClose={() => setSwapOrder(null)} />
Conditionally rendered (the dialog itself handles open state via order != null).
Lets schedulers reassign a visit to a different team while keeping the time slot.
Fully gated by calendar.swap-teams permission — without it, the swap button never appears in the visit block, so this dialog never opens.

12. EDGE CASES & GOTCHAS
activeDivisionIds starts as null so the first-render useMemo returns "all divisions". The first chip click migrates it from null → real Set. Prevents the calendar from being empty during division load.
No URL state sync — refreshing the page resets date to today and all filters to ON. Filters are session-local.
Date navigation has no calendar popup — only ±1 day arrows. Jumping weeks ahead requires repeated clicks (or future enhancement: add a date picker).
Filter chips wrap with flex-wrap — on narrow viewports the toolbar grows vertically. No horizontal scroll fallback.
showAllTeams is hardcoded true here — this differs from CreateVisit which only shows skill-matching teams.
hideHeader is hardcoded true — the page provides its own date toolbar; TeamCalendar's built-in header would duplicate it.
Two visit types share warning color (site_visit_single & site_visit_contract) — visually indistinguishable from the chip alone; differentiated only by label.
No "select all / clear all" filter buttons — toggling 8 chips one-by-one is the only way to reset.
