Core Logic Files
File	Lines	What It Does
orderStateMachine.ts	23	Defines valid status transitions. canTransition("scheduled", "confirmed") → true. Terminal states: completed, cancelled
useOrders.ts	~164	Hook that fetches all orders with joins to customers, phones, order_details, visits, teams
useQuotations.ts	—	Hook that fetches quotations + provides removeApproval(), cancelQuotation(), convertToOrder() actions
useFollowUpSource.ts	—	Hook that loads a source order's services with follow-up flags for the Create Follow-Up page
useDeductOrderStock.ts	41	FIFO stock deduction mutation — when an order is confirmed, deducts inventory from oldest stock layers first
promotionEngine.ts	—	Calculates campaign + voucher discounts (per-line and total)
subscriptionEngine.ts	—	Calculates subscription package discounts
bookingEngine.ts	—	Auto-assign algorithm: finds available team+time slots matching service skills
Status Transitions (State Machine)
tentative       → scheduled, cancelled
scheduled       → confirmed, cancelled, waitlist, pending-confirmation
pending-confirm → confirmed, scheduled, cancelled
confirmed       → in-progress, cancelled, scheduled
in-progress     → completed, cancelled
completed       → (terminal — no transitions)
cancelled       → (terminal — no transitions)
waitlist         → scheduled, cancelled
pending-approval → confirmed, cancelled
That's everything — every page, dialog, tab, button, filter, input, and action in the entire Orders module.
