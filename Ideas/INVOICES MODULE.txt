HOOKS
useInvoices() → ["invoices"]
Calls mark_overdue_invoices RPC first (ensures statuses are current)
Fetches from invoices with joins: customers!inner → customer_phones, payments (filtered by deleted_at IS NULL), invoice_line_items
Maps DB snake_case to UI camelCase
Limit: 500 records
usePayments() → ["payments"]
Fetches payments (limit 500, soft-delete filtered)
Collects unique invoice_ids → batch fetches invoices
Collects unique customer_ids → parallel fetches customers + customer_phones (primary only)
Builds lookup maps for O(1) resolution
usePendingPayments() → ["pending-payments"]
Fetches invoices where status NOT IN (void, paid, cancelled) with nested customer/phone
Filters out invoices where remaining ≤ 0
Groups by customer_id into CustomerPending[]
No limit (fetches all pending)
DATABASE TABLES USED
Table	Usage
invoices	Main invoice data, status updates, void
invoice_line_items	Line items per invoice

activity_log	Audit trail for all actions

ONLINE PAYMENT FLOW (Dibsy)
File: supabase/functions/_shared/finalizePayment.ts

When an online payment completes:

Checks payment_sessions.receipt_sent (idempotent guard)
Loops through invoice_allocations in the session
For each allocation: inserts payments record with auto-generated PAY-{timestamp}-{uuid} ID
Calls apply_payment RPC (atomic paid_amount update on invoice — prevents race conditions)
Marks receipt_sent = true on session
Writes audit log entry
